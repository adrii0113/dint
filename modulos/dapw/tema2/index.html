Servidores de ficheros

¿Ques son los servidores de ficheros?

Un file server (o servidor de archivos) es un servidor central de una red de ordenadores, que pone a disposición del cliente conectado un conjunto de archivos o parte del mismo. De este modo, el servidor de archivos proporciona a los usuarios un lugar de almacenamiento centralizado para los archivos en sus propios soportes de datos, disponible para todos los clientes autorizados. El administrador del servidor establece unas directrices estrictas para determinar qué usuarios tienen derechos de acceso y a qué datos: por ejemplo, configurando el sistema u otorgando los permisos del sistema de archivos, se puede determinar qué carpetas puede ver y abrir un usuario o grupo de usuarios concreto y si los archivos solo son visibles o también pueden guardarse, modificarse o eliminarse.

Servidor FTP

Servicio orientado a conexion, se creo en las decada de lo s70' utiliza los puerto 20 y 21 por defecto sobre tcp, el 20 para datos y 21 para control. nos permite realizar intercambio de archivos de forma independiente de los so, da igual el so, estariamos trabajando con una arquitectura cliente servidor y no tiene segurirdad pero podemos añadirla con certiicados. El servicio cuenta tanto con parte cliente como con parte servidor, en la parte de cliente podemos encontrar FileZilla y en la parte servidor FileZilla Server. Ejemplos de servidores FTP:

Ejemplos de servidore ftp: proftp,vsftp y filezilla. Ejemplos de clientes: navegador, cliente modo texto(ftp), cliente grafico(filezilla, winstp, cyberduck).

Permisos en FTP

Dentro de los permisos de FTP podemos encontrar tres niveles de acceso:

Nivel de acceso R: nos permite listar el contenido del directorio actual

Nivel de acceso w: nos permite escribir en el contenido del fichero actual

Nivel de acceso x: nos permite ejecutar

Permisos del lado del cliente

Open <IP>: abre la conexión

user <usuario>: solicita un usuario para utenticar

bye/exit/quit: salir

close: cerrar conexión sin salir

mkdir: crear carpeta en destino (si tenemos permisos)

!mkdir: crear carpeta en origen (si tenemos permisos)

get <archivo>: solicita archivo remoto

put <archivo>: transfiere un archivo al servidor remoto

user <usuario>: solicita un usuario para utenticar

pwd: ruta destino actua

!pwd: ruta origen actual

ls: directorio destino actual

!ls: directorio origen actua

rmdir: eliminar carpeta en destino (si tenemos permisos)

!rmdir: eliminar carpeta en origen (si tenemos permisos)

mget <archivo>: recupera la lista (con patron) de archivos remotos

Tipos de usuarios en un servidor FTP

Exixsten tres tipos de usuarios en un servidor FTP: los anonimos, los autenticados y los virtuales

Usuarios anonimos:

Los anonimos usan como nombre anonymous, siempre acceden con permisos limitados, como pass utilizan un email,y si permitimos este tipo de user tenemos que tener en cuneta la posible brecha de segurirdad asociad. Los utilizaremos cuando quereamos exponer al publico ciertos ficheros.

Usuarios autenticados

Los autenticados o usuarios del sistema: estan creados en el propio s.o y tienen nombre y pass propios

Usuarios virtuales

son creados exclusivamente para servicios FTP, son el tipo de user que crea menor brecha de seguridad.La validacion del par username y pass puede realizarse mediante ficheros, bases de datos o servicios de directorios(ldap).

Tipos de ficheros en un servidor FTP

Existen varios tipos de ficheros: los binarios y de texto

Ficheros binarios:

No contienen caracteres legibles por humanos

Ficheros de texto:

Conrtienen caracteres legibles por humanos

Modos de conexion en un servidor FTP

Hay dos tipos de modos de conexion, el modo activo y el modo pasivo

Modo activo:

es el modo por defecto, en este modo el servidor abre los puertos 20 para datos y 21 para control, el cliente hace lo mismo pero con puertos superiores a 1024. si vamos a subir al server la conexion la inicia el cliente, si vamos a descaragr un fichero del server la concexion la inicia el servidor.

Modo pasivo:

es el cliente el que siempre inicia las conexiones, tato la de dato como la de control. Nos evita problemas si existen cortafuegos entre medias.

Errores en un servidor FTP

Dependiendo del patron numerico del error puede significar una cosa u otra

22x --> conexion correcta

120 --> El servidor lee en x minutos

331 --> Nombre de usuario correcto, necesita una contraseña

550 --> Solicitud de información no aceptada. Las filas no están disponibles (ej.: filas incorrectas).

22x --> conexion correcta

Directivas FTP

Server name:

Esta directiva se utiliza para cambiar el nombre del servidor

Server Type:

Esta directiva se utiliza para configurar el modo del servidor, ya sea como servicio o como programa

Default Root:

Establece la ruta de la carpeta inicial de un usuario, con el caracter (~) podemos enjaular al usuario para que no recorra el sistema de ficheros

Max clients:

Esta directiva se utiliza para establecer el numero maximo de usuarios que puede haber en el servidor

User alias:

Esta directiva se utiliza para establecer el alias del usuario del sistema

Display login:

Esta directiva se utiliza para mostrar el mensaje de bienvenida de login
